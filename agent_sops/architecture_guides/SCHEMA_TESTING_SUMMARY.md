## Schema Verification

### ✅ Foreign Key Schema (CORRECT)

**Experiments:**
```javascript
{
  experiment_id: "...",
  progress: {
    trajectories_loaded: 1  // Count only!
  }
  // ⚠️  NO trajectory_refs array
}
```

**Trajectories:**
```javascript
{
  trajectory_id: "...",
  experiment_id: "..."  // ← Foreign key to experiments
}
```

**Query Pattern:**
```javascript
// O(1) with index
db.trajectories.find({ experiment_id: "..." })
  .skip(0)
  .limit(100)  // Pagination!
```

### ❌ Array Schema (WRONG - Not Used)

```javascript
// This would be WRONG (not in our tests):
{
  experiment_id: "...",
  trajectory_refs: ["traj_1", "traj_2", ...]  // ❌ 16MB limit!
}
```

---

## Performance Characteristics

| Operation | Array Approach | Foreign Key Approach |
|-----------|---------------|---------------------|
| **Add trajectory** | O(n) array append | O(1) insert |
| **Query trajectories** | O(1) load + O(n) iterate | O(1) with index |
| **Pagination** | Manual slicing | Native skip/limit |
| **Max trajectories** | ~400K (16MB limit) | Unlimited |
| **Memory usage** | Load all IDs | Load only queried docs |
